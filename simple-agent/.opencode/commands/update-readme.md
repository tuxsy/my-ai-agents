---

Analiza el proyecto y actualiza el archivo README.md con la información más reciente.

El README debe incluir las siguientes secciones:

1. **Título y descripción** - Qué es el proyecto y qué hace
2. **Requisitos** - Versión mínima de Python requerida (revisa las dependencias en requirements.txt para determinarla)
3. **Tecnologías** - Stack técnico utilizado
4. **Instalación** - Pasos para instalar dependencias y configurar el proyecto
5. **Uso** - Cómo ejecutar el agente
6. **Nota para desarrolladores** - Explicar que se usa pigar para mantener requirements.txt y cómo usarlo

Mantén el README conciso y en español. No uses emojis. Cambia el README sólo si hay información nueva o desactualizada, no hagas cambios innecesarios.

Si has realizado algún cambio en el README (y sólo si has realizado algún cambio), indicalo al final del documento con un texto como "Modificado por {info_modelo}". Proporciona la información mínima necesaria para identificar el modelo usado, incluyendo si ha usado razonamiento o no.
